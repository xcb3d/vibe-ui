Accordion

A vertically stacked set of interactive headings that each reveal a section of content. Built for tough interfaces with zero compromises on style.

API Reference Source Code
01. Is it accessible? expand_more

Yes. It adheres to the WAI-ARIA design pattern. You can navigate through the items using keyboard shortcuts.

02. Can I style it? expand_more

Absolutely! It comes with unstyled primitive classes that you can override with your own CSS or utility classes.

03. Is it animated? expand_more

Yes, but you can disable animations easily. We use CSS transitions for smooth height adjustments.

# Installation

terminal
npm install @neubrutal/accordion

# Usage 

import {
  Accordion,
  AccordionContent,
  AccordionItem,
  AccordionTrigger,
} from "@/components/ui/accordion"
export function AccordionDemo() {
  return (
    <Accordion type="single" collapsible>
      <AccordionItem value="item-1">
        <AccordionTrigger>Is it accessible?</AccordionTrigger>
        <AccordionContent>
          Yes. It adheres to the WAI-ARIA design pattern.
        </AccordionContent>
      </AccordionItem>
    </Accordion>
  )
}

API Reference

The accordion is built using the basic HTML details/summary pattern but enhanced with Radix UI primitives for full accessibility.

Prop Type Default Description
type "single" | "multiple" "single" Determines whether one or multiple items can be opened at the same time.
collapsible boolean false Allows closing all items when in single mode.
defaultValue string | string[] - The value of the item that should be open by default.
disabled boolean false Prevents interaction with the accordion item.
